# Éviter les interstitiels et les boîtes de dialogue intrusifs

 
 
 
 Les interstitiels et les boîtes de dialogue intrusifs sont des éléments de page qui empêchent les utilisateurs de voir le contenu de la page, généralement à des fins promotionnelles. Les interstitiels couvrent la page entière, tandis que les boîtes de dialogue n'en couvrent qu'une partie, cachant aussi parfois le contenu qui se trouve dessous.

 
 Les sites Web ont fréquemment besoin d'afficher des boîtes de dialogue pour diverses raisons. Cependant, interrompre les utilisateurs avec des interstitiels intrusifs peut les frustrer et nuire à la confiance qu'ils accordent à votre site Web.

 
 Les boîtes de dialogue et les interstitiels intrusifs empêchent Google et d'autres moteurs de recherche de comprendre votre contenu, ce qui peut nuire aux performances dans la recherche. De même, si les utilisateurs trouvent votre site difficile à utiliser, il seront peu enclins à y revenir, y compris via des moteurs de recherche.

# Créer des boîtes de dialogue non intrusives

 
 Créer des boîtes de dialogue non intrusives permet aux utilisateurs d'accéder à votre contenu sans être gênés par une boîte de dialogue. Cela concerne tous les types de boîtes de dialogue promotionnelles, y compris les invites d'installation d'application. Le respect de ces bonnes pratiques permettra à vos utilisateurs de profiter d'une expérience agréable sur votre site et aidera la recherche Google à comprendre le contenu et la structure de votre site.

# Utiliser des bannières plutôt que des interstitiels

 

 
 Plutôt que des interstitiels pleine page, utilisez des bannières qui n'occupent qu'une petite partie de l'écran pour attirer l'attention des utilisateurs. Les bannières permettent aux utilisateurs et aux moteurs de recherche d'accéder au contenu dès qu'ils arrivent sur la page.

 
 Vous pouvez les implémenter de différentes manières. Par exemple, pour les bannières incitant à installer une application, vous pouvez utiliser une bannière compatible avec le navigateur, comme une bannière Smart App pour Safari ou une bannière d'installation dans l'application pour Chrome. Une autre option consiste à créer une bannière HTML simple, semblable à une petite publicité ordinaire, qui renvoie vers la bonne plate-forme de téléchargement d'applications. Vous pouvez réutiliser ces petits conteneurs pour d'autres types de notifications, comme les invites d'inscription à une newsletter.

# Utiliser des bibliothèques courantes

 
 De nombreux CMS incluent des plug-ins pour créer des boîtes de dialogue et des interstitiels standards pour les cas d'utilisation les plus courants, tels que les invites d'inscription à une newsletter. Par exemple, si vous utilisez WordPress, recherchez "inscription à une newsletter wordpress". L'utilisation de ces plug-ins peut être utile pour Google, d'autres moteurs de recherche et Internet en général, car les développeurs de plug-ins peuvent déployer des améliorations à grande échelle.

# Éviter les erreurs fréquentes

 
 Sauf obligation légale, évitez les erreurs courantes suivantes lorsque vous concevez une boîte de dialogue ou un interstitiel afin d'aider la recherche Google à explorer et à comprendre votre contenu :

 
 
- Ne masquez pas toute la page avec des interstitiels. 
 
- Ne redirigez pas l'utilisateur vers une page distincte pour obtenir son consentement ou ses commentaires. 

# Interstitiels obligatoires

 
 En raison du type de contenu qu'ils publient, certains sites sont tenus d'afficher un interstitiel. Par exemple, un site de casino peut avoir besoin d'afficher une page de vérification de l'âge, qui est un type d'interstitiel où l'utilisateur doit indiquer son âge avant d'accéder au contenu.

 
 Les interstitiels obligatoires ne sont pas concernés par les consignes décrites dans ce document. Toutefois, nous vous recommandons de respecter les bonnes pratiques suivantes, si possible :

 
 
- 
 Assurez-vous que le contenu est simplement recouvert par l'interstitiel. De cette façon, Google peut indexer au moins une partie du contenu et l'afficher éventuellement dans les résultats de recherche.
 
 
- 
 Ne redirigez pas les requêtes HTTP entrantes vers une autre page pour recueillir le consentement de l'utilisateur ou fournir des données. Si vous redirigez toutes les URL vers une seule page, toutes les pages sauf celle-ci seront supprimées des résultats de recherche, car Googlebot ne peut extraire que cette page.
 

 
 Concernant le contenu réservé aux adultes, nous vous recommandons d'autoriser l'exploration de votre contenu par Googlebot sans déclencher la limite d'âge. Pour ce faire, vous pouvez valider les requêtes Googlebot et présenter le contenu sans limite d'âge.
